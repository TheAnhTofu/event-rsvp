# Memory – Event RSVP

Memory lưu **quyết định** và **thuật ngữ** để agent và dev nhất quán theo thời gian.

## Khi nào cập nhật

- **decisions.md:** Kiến trúc, chọn DB/auth/email provider, đổi API, trade-off quan trọng. Ghi ngày (YYYY-MM-DD).
- **glossary.md:** Thuật ngữ domain mới hoặc định nghĩa đổi.
- **product-backlog.md:** Ticket ưu tiên từ stakeholder / PM (copy, bug, admin); có ID `TICKET-YYYY-MM-DD-nn` và acceptance ngắn.

## Format

- **decisions:** Context → Decision → Consequences (ADR ngắn).
- Không xóa lịch sử; nếu đảo ngược quyết định, thêm entry mới và ghi “Supersedes …” nếu cần.
